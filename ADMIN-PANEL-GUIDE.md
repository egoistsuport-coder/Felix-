## 💾 ХРАНИЛИЩЕ ДАННЫХ:

### Текущее решение: In-Memory (Map)
- Данные хранятся в памяти сервера
- При перезапуске данные теряются
- Подходит для тестирования

### Для продакшена рекомендуется:
1. **База данных** (PostgreSQL, MongoDB)
2. **Supabase** (готовое решение)
3. **Vercel KV** (Redis)
4. **JSON файлы** (простое решение)

### Миграция на базу данных:
Заменить Map на запросы к БД в `api/admin.js`:
```javascript
// Вместо
partnerRequests.set(requestId, request);

// Использовать
await db.partnerRequests.create(request);
```

## 📱 ИНТЕГРАЦИЯ С БОТОМ:

### Добавить команду /admin:
Открыть `api/webhook.js` и добавить:
```javascript
if (cmd === '/admin') {
    const isAdmin = userId === 8264612178;
    if (!isAdmin) {
        await send(chatId, '❌ Доступ запрещен');
        return;
    }
    
    const buttons = [[{
        text: '🛡️ Открыть админ-панель',
        web_app: { url: 'https://felix-black.vercel.app/miniapp/admin.html' }

    await send(chatId, '🛡️ <b>Админ-панель Felix Bot</b>\n\nУправление курсами, партнерами и заявками', buttons);
}
```

### Уведомления о новых заявках:
Добавить в `api/admin.js` функцию отправки уведомления:
```javascript
async function notifyAdmin(message) {
    await fetch(`https://api.telegram.org/bot${TOKEN}/sendMessage`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
            chat_id: 8264612178,
            text: message
        })
    });
}

// В handleSubmitPartnerRequest:
await notifyAdmin(`📝 Новая заявка на партнерство!\n\n${name}\n${url}`);
```

## 📊 API ENDPOINTS:

### Публичные:
- `submitPartnerRequest` - подать заявку
- `getCourses` - получить курсы
- `getPartners` - получить партнеров

### Админские (требуют userId в ADMIN_IDS):
- `isAdmin` - проверка прав
- `getPartnerRequests` - список заявок
- `approvePartnerRequest` - одобрить заявку
- `rejectPartnerRequest` - отклонить заявку
- `addCourse` - добавить курс
- `updateCourse` - обновить курс
- `deleteCourse` - удалить курс
- `addPartner` - добавить партнера
- `updatePartner` - обновить партнера
- `deletePartner` - удалить партнера
