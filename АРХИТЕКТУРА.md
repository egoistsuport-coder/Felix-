## 🔄 Поток данных

### 1. Текстовое сообщение
```
User → Telegram → Webhook → webhook.js
                              ↓
                         Add to history
                              ↓
                         getAIResponse()
                              ↓
                         Groq API (LLaMA 3.3)
                              ↓
                         Response
                              ↓
                         Add to history
                              ↓
                         Send to user
```

### 2. Голосовое сообщение
```
User → Telegram → Webhook → webhook.js
                              ↓
                         downloadVoiceFile()
                              ↓
                         transcribeVoice()
                              ↓
                         Groq API (Whisper)
                              ↓
                         Transcription
                              ↓
                         Add to history
                              ↓
                         getAIResponse()
                              ↓
                         Groq API (LLaMA 3.3)
                              ↓
                         Response
                              ↓
                         Send to user
```

### 3. Команда /summary
```
User → /summary → webhook.js
                     ↓
                Check history (min 5 messages)
                     ↓
                createSummary()
                     ↓
                Groq API (LLaMA 3.3)
                     ↓
                Summary
                     ↓
                Send to user
```

### 4. Команда /analyze
```
User → /analyze <text> → webhook.js
                            ↓
                       Validate (min 10 words)
                            ↓
                       analyzeText()
                            ↓
                       Groq API (LLaMA 3.3)
                            ↓
                       Analysis
                            ↓
                       Send to user
```

### 5. Команда /generate
```
User → /generate <prompt> → webhook.js
                               ↓
                          generateContent()
                               ↓
                          Groq API (LLaMA 3.3)
                               ↓
                          Generated content
                               ↓
                          Send to user
```

## 🔧 Компоненты

### api/webhook.js
**Назначение:** Основной обработчик всех запросов от Telegram

**Функции:**
- `sendMessage()` - Отправка сообщений в Telegram
- `downloadVoiceFile()` - Скачивание голосовых файлов
- `transcribeVoice()` - Транскрипция голоса (Whisper)
- `getAIResponse()` - Получение AI ответов (LLaMA)
- `organizeText()` - Организация текста
- `createSummary()` - Создание саммари
- `analyzeText()` - Анализ текста
- `generateContent()` - Генерация контента

**Хранилище:**
- `conversations` - Map<userId, messages[]>
- `userSettings` - Map<userId, settings{}>

**Команды:**
- `/start` - Главное меню
- `/summary` - Саммари диалога
- `/analyze` - Анализ текста
- `/generate` - Генерация контента
- `/organize` - Структурирование
- `/clear` - Очистка истории

### miniapp/index.html
**Назначение:** Веб-интерфейс для демонстрации возможностей

**Секции:**
- Новые возможности v4.0
- Команды с описаниями
- Как использовать
- Технологии
- Примеры использования

**Технологии:**
- Vanilla JavaScript
- Telegram WebApp API
- CSS3 (градиенты, анимации)

## 📈 Масштабирование

### Текущая версия (Simple)
**Ограничения:**
- In-memory storage (данные теряются при рестарте)
- Нет постоянной истории
- Нет поиска по истории
- Нет тегов и категорий

**Преимущества:**
- Быстрый старт (1 минута)
- Минимум зависимостей
- Все AI функции работают
- Не требует настройки БД

### Будущая версия (Full)
**Возможности:**
- Постоянное хранение в PostgreSQL
- История всех диалогов
- Поиск по истории
- Теги и категории
- Экспорт в различные форматы
- Статистика использования
- Настройки пользователя

**Требования:**
- Supabase аккаунт
- Настройка БД (10 минут)
- Дополнительные environment variables

## 🚀 Производительность

### Текущие показатели
- **Время ответа:** ~1-3 секунды
- **Транскрипция голоса:** ~2-5 секунд
- **Генерация контента:** ~3-7 секунд
- **Создание саммари:** ~2-4 секунды

### Оптимизация
- ✅ Использование Groq API (быстрый inference)
- ✅ In-memory storage (быстрый доступ)
- ✅ Serverless функции (автомасштабирование)
- ✅ Контекст ограничен 10 сообщениями

## 🔄 Обновление до Full версии

### Шаги
1. Настройте Supabase (10 минут)
2. Выполните `database/v4-schema.sql`
3. Добавьте environment variables
4. Обновите `api/webhook.js` для использования `lib/db.js`
5. Подключите дополнительные API endpoints

### Инструкция
См. `DEPLOY-V4.md` и `database/SETUP-SUPABASE.md`
